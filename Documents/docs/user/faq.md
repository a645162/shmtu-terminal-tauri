# 常见问题

## 为什么没有账单

先依次检查：

1. 是否已选择身份——底部状态栏显示的是哪个身份？
2. 身份下是否真的挂了账号——去"身份与账号管理"里确认
3. 账号是否启用了同步——检查账号的 `enable_update` 开关
4. 是否已经执行过同步——没有同步过当然没有数据

如果以上都正常但仍无数据，尝试做一次增量同步。

## 为什么一直要验证码

这通常不是异常，而是你当前选择了手动验证码模式。手动模式下，每次登录都需要你亲自输入验证码。

建议：

1. 先用手动验证码完成一次同步，确认账号密码正确
2. 再到设置中切换到 OCR 模式（远程 OCR 或本地 ONNX）
3. 切换后先去"验证码测试"确认 OCR 能正常识别

## 远程 OCR 连接失败

检查以下几点：

- OCR 服务是否正在运行
- 设置中的 OCR 地址和端口是否正确（TCP 模式需要填 host 和 port，HTTP 模式需要填完整 URL）
- 默认 HTTP OCR 地址是 `http://127.0.0.1:5000`，如果 OCR 服务在远程服务器上，需要修改为实际地址
- 网络是否通畅，防火墙是否放行

## 本地 ONNX 模式无法工作

优先检查：

- 模型是否完整下载——去"验证码测试"页面查看模型状态
- 模型路径是否正确——设置中可以查看和修改 ONNX 模型路径
- 如果已经出现模型损坏提示，最稳妥的方式是删除本地模型并重新下载

## 为什么统计看起来不对

通常先查同步，再查重复，再查时间范围：

1. 历史同步是否完整——有没有只同步了最近几天的数据？
2. 是否有重复账单——去账单页做一次去重
3. 分类规则是否需要更新——如果大量显示"未分类"，说明规则文件可能缺失或过时
4. 你看的时间范围是否正确——确认统计页选的时间段和你预期的一致

不要直接把统计异常等同于原始数据损坏。

## 切换身份后内容没有刷新

通常是以下原因之一：

- 身份没有真正切换成功——检查底部状态栏是否已变更为目标身份
- 账单列表仍停留在旧分页条件——切换后手动刷新一下账单页
- 账号列表还未重新加载——关闭并重新打开账单页

## 导入后出现重复记录

这是典型的导入后整理问题，不一定是导入失败。建议流程：

1. 先做账号级或身份级去重
2. 必要时执行重建
3. 重新核对统计结果

## 快照恢复后程序行为和之前不一致

先确认恢复的快照时间点是否正确。快照恢复的不只是账单，还可能包含配置和状态变化。如果快照是在不同配置下创建的，恢复后配置也会回退到当时的值。

## 为什么我不敢点恢复

如果你先做了快照，就没那么危险。快照的作用就是让你可以随时退回到操作前的状态。建议：

- 大操作前必做快照
- 恢复完先检查首页和账单页
- 如果恢复结果不满意，可以用之前的快照再恢复一次

## 同步报错"用户名或密码错误"

检查账号配置中的账号 ID 和密码是否正确。密码是加密存储的，如果之前手动改过数据库文件可能需要重新在程序中填写密码。

## 程序启动很慢

可能的原因：

- 启用了启动密码保护，每次需要输入密码
- 首次启动需要初始化数据目录和下载规则文件
- 本地 ONNX 模型加载需要时间

## 最重要的建议

不会的设置先别动，先把账单同步出来。大部分问题都可以通过"先做快照，再尝试操作"来解决。
