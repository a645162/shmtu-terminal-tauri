# 怎么同步账单

大多数时候，你只需要记住这三种同步：

- `增量同步`：平时更新最近账单
- `全量同步`：补齐历史数据
- `单账号同步`：只同步某一个账号

## 增量同步——最常用的方式

增量同步是日常使用最频繁的操作，它只抓取最近可能新增的账单数据。

特点：

- 速度快，通常几秒到几十秒就能完成
- 只抓取最近的新增数据，对已同步数据有早停机制（遇到连续旧数据就停止翻页）
- 适合每天或每周顺手更新一次

什么时候用：

- 每天打开软件顺手更新一次
- 每周补一次最近记录
- 自动同步默认使用的就是增量模式

## 全量同步——补齐历史的手段

全量同步会清除该身份下的旧数据，然后重新从校园平台拉取。

特点：

- 覆盖范围更大，默认最大翻页深度 1000 页
- 耗时更久，取决于历史数据量
- 会清除旧的合并账单和操作日志，重新来过

什么时候用：

- 第一次使用，想尽量把历史数据补全
- 历史数据明显不全，怀疑以前有漏同步
- 需要重新补一段较久以前的记录

::: warning 注意
全量同步会先清除该身份下的旧合并数据和原始数据，再重新拉取。虽然原始数据是重新获取的，但操作前建议先做一次快照。
:::

## 单账号同步——只修一个账号

如果一个身份下挂了多个账号，而只有其中一个有问题，可以只同步那个账号，不用把整组都重跑。

适用场景：

- 某个账号同步报错，其他账号正常
- 只想更新某一个账号的近期数据
- 新增了一个账号，只想同步它

## 同步范围怎么选

同步时可以选择时间范围，范围越大耗时越长：

| 范围 | 覆盖时段 | 适合场景 |
|------|---------|---------|
| 最近一周 | 7 天 | 快速更新 |
| 最近半个月 | 15 天 | 补近期数据 |
| 最近一个月 | 30 天 | 日常更新 |
| 最近半年 | 183 天 | 补历史数据 |
| 最近一年 | 365 天 | 大范围补数据 |
| 全部 | 不限 | 完整重建 |

最简单的用法：

- 平时更新：`最近一个月`
- 补历史数据：`最近半年`
- 大修复：`全部`

## 验证码怎么选

同步过程中需要登录校园平台，登录时需要验证码。验证码模式决定了登录方式：

### 手动验证码（推荐首次使用）

- 每次需要你自己输入验证码图片上的文字
- 最稳定，最适合首次验证环境
- 缺点是每次都需要人工操作

### 远程 OCR（TCP）

- 需要一个运行中的远程 OCR TCP 服务
- 程序自动发送验证码图片到远端识别
- 优点是省事，缺点是依赖外部服务

### 远程 OCR（HTTP）

- 需要一个运行中的远程 OCR HTTP 服务（默认地址 `http://127.0.0.1:5000`）
- 通过 RESTful 接口识别验证码
- 同样依赖外部服务，但比 TCP 方式更通用

### 本地 ONNX OCR

- 在本机加载 ONNX 模型自动识别验证码
- 优点是不依赖远端服务，完全本地完成
- 缺点是第一次需要下载模型（约几十 MB）

推荐顺序：首次使用先选手动 → 确认同步正常 → 再切换到 OCR 模式。

## 同步过程中会看到什么

同步不是一瞬间完成的，你会看到这些阶段依次出现：

1. **检查登录状态**：看看之前的登录是否还有效
2. **获取验证码**：需要验证码时会弹出验证码图片
3. **登录**：提交验证码和密码
4. **分页拉取账单**：逐页获取消费记录
5. **写入原始数据**：把拉到的记录存入数据库
6. **合并到身份账单**：把原始账单合并到身份级别的展示层

如果同步停在"需要验证码"，通常不是卡死，而是在等你输入验证码。输入后点继续，同步会自动续传，不会从头再来。

## 常见问题

**同步一直要求验证码怎么办？**

如果你选了手动验证码，这就是正常的。如果想自动识别，可以切换到 OCR 模式。

**同步后没有新数据？**

可能是你选的范围太小，或者最近确实没有新消费。试试扩大范围。

**同步报错"用户名或密码错误"？**

检查账号配置中的账号 ID 和密码是否正确。密码是加密存储的，如果之前手动改过数据库可能需要重新填写。

## 最简单的建议

- 平时更新：增量同步
- 补历史：全量同步
- 某个账号异常：单账号同步
- 第一次使用：先手动验证码
