# 怎么备份和恢复

如果你要导入数据、恢复旧数据、或者准备做大调整，先看这页。

## 最重要的一条

**做大操作前，先创建快照。**

快照是程序内置的”存档”机制，创建快照后，即使后续操作出了问题，也可以通过恢复快照退回来。

## 什么情况下要先备份

建议在这些操作前先做一次快照：

- 导入数据（导入后可能出现重复，需要清理）
- 大范围清理数据（删除大量账单）
- 尝试恢复旧内容（恢复可能覆盖当前数据）
- 重新整理大量账单（去重、重建）
- 更换分类规则文件（规则变化会影响展示层）
- 全量同步前（全量同步会清除旧数据）

## 快照有什么用

你可以把快照理解成”给当前状态拍一张存档”。它不是只备份某一张表，而是把程序关键数据整体打包，包括：

- 原始账单数据
- 合并账单数据
- 身份与账号信息
- 会话信息
- 操作日志

如果后面操作结果不满意，还可以通过恢复快照退回来。

::: tip 提示
程序默认保留最近 10 个快照（可在设置中调整 `snapshot_keep_count`），超过数量会自动清理最旧的。
:::

## 导出适合什么时候用

导出和快照不同，导出是把账单数据转成外部格式，适合：

- 想自己留一份数据（比如保存到 U 盘）
- 想在 Excel 或其他表格软件里查看分析
- 想导入到其他记账工具

支持的导出格式：

| 格式 | 用途 |
|------|------|
| CSV | 表格查看和简单分析 |
| JSON | 备份与程序间数据交换 |
| 钱迹 | 导入到”钱迹”记账 App |

## 恢复后要检查什么

恢复快照后，不要以为就万事大吉了。建议马上检查三件事：

1. **当前身份对不对**：底部状态栏显示的身份名称是否正确
2. **账单数量对不对**：去账单页看看记录数量是否和快照时一致
3. **最近统计看起来正不正常**：首页的消费金额是否合理

::: warning 注意
快照恢复的不只是账单，还可能包含配置和状态变化。如果快照是在不同配置下创建的，恢复后配置也会回退到当时的值。
:::

## 数据导入注意事项

当前导入主要面向 JSON 格式的数据。导入前建议固定做三步：

1. 先做快照（给自己留退路）
2. 确认目标身份（导入数据会归入当前选中的身份）
3. 导入后执行一次去重或重建检查（防止重复记录）

## 最简单的习惯

- 动数据前先快照
- 恢复完先检查
- 不确定的操作，先导出一份数据再动手
