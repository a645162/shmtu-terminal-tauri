# 设置与数据

这一页不追求把每个字段讲成技术说明，而是帮你判断：哪些设置要动，哪些设置不要乱动。

## 大部分用户最常碰到的是这几类设置

设置页大致分成：

- 安全
- 身份
- 验证码
- 同步
- 数据
- 分类
- 更新
- UI
- 调试

## 安全：这台电脑是不是只有你自己用

### 启动保护

启用后，程序启动时会要求输入启动密码。

适合：

- 多人共用设备
- 本地保存了真实账号信息

不太需要开的情况：

- 完全离线个人电脑且你更在意打开速度

## 验证码：先选一个你能稳定用的方式

你主要会看到这些内容：

- 验证码模式
- 远程 OCR 地址
- HTTP OCR 地址
- OCR 重试次数
- 本地 ONNX 模型路径

实际建议：

- 远程服务不稳定时，降低自动依赖，回退到手动模式
- OCR 连续误判时，先做验证码测试，不要直接怀疑同步逻辑

## 同步：大部分时候不用细调

这里会看到一些同步参数：

- `max_pages`
- `early_stop_threshold`
- `skip_graduated_accounts`
- `auto_merge_after_sync`
- `auto_sync_enabled`
- `auto_sync_interval_minutes`
- `auto_sync_range`

对普通用户来说，理解到这个程度就够了：

- `max_pages` 决定单次同步最大翻页深度
- `early_stop_threshold` 决定遇到旧数据时何时提前停止
- `auto_sync_range` 决定自动同步的默认时间窗口

如果你不确定，就先保持默认值，不要一开始就把范围和页数调得很大。

## 分类：影响的是“怎么看”，不是“有没有账单”

分类相关设置主要影响：

- 规则文件路径
- 规则更新地址

它们影响的是“展示解释层”，不是原始账单是否入库。

## 数据导出：想留存、分析或迁移时再用

支持格式：

- `CSV`
- `JSON`
- `钱迹`

适合这样用：

- `CSV`：适合表格查看和简单分析
- `JSON`：适合备份与程序间交换
- `钱迹`：适合外部记账生态导入

## 数据导入：动手前先备份

当前导入主要面向 `JSON` 数据。

导入前建议固定做这三步：

1. 先做快照
2. 确认目标身份
3. 导入后执行一次去重或重建检查

## 数据快照：出手前先给自己留后路

快照是最重要的保险措施之一。

下面这些操作前，建议都先建快照：

- 大规模导入
- 手动清理数据
- 更换规则文件
- 恢复历史账单
- 尝试全量重建

恢复快照后，记得重新检查：

- 当前身份是否正确
- 账单数量是否符合预期
- 设置是否也回到了目标版本

## 更新与调试

### 更新

应用支持检查新版本，但是否升级仍建议由你自行判断。

### 调试

调试页主要是给排错用的。普通用户如果只是日常同步和看账单，尽量不要在这里做恢复类操作。
