# 设置与数据

本页说明设置项含义，以及导出、导入、快照的使用建议。

## 设置分类

程序设置大致分为：

- 安全
- 身份
- 验证码
- 同步
- 数据
- 分类
- 更新
- UI
- 调试

## 安全设置

### 启动保护

启用后，程序启动时会要求输入启动密码。

适合：

- 多人共用设备
- 本地保存了真实账号信息

不适合：

- 完全离线个人电脑且你更在意打开速度

## 验证码设置

核心关注项：

- 验证码模式
- 远程 OCR 地址
- HTTP OCR 地址
- OCR 重试次数
- 本地 ONNX 模型路径

建议：

- 远程服务不稳定时，降低自动依赖，回退到手动模式
- OCR 连续误判时，先做验证码测试，不要直接怀疑同步逻辑

## 同步设置

重点字段：

- `max_pages`
- `early_stop_threshold`
- `skip_graduated_accounts`
- `auto_merge_after_sync`
- `auto_sync_enabled`
- `auto_sync_interval_minutes`
- `auto_sync_range`

建议理解方式：

- `max_pages` 决定单次同步最大翻页深度
- `early_stop_threshold` 决定遇到旧数据时何时提前停止
- `auto_sync_range` 决定自动同步的默认时间窗口

如果你不确定怎么调，保守地维持默认值通常比手动激进放大更稳。

## 分类设置

分类相关设置主要影响：

- 规则文件路径
- 规则更新地址

它们影响的是“展示解释层”，不是原始账单是否入库。

## 数据导出

支持格式：

- `CSV`
- `JSON`
- `钱迹`

适用场景：

- `CSV`：适合表格查看和简单分析
- `JSON`：适合备份与程序间交换
- `钱迹`：适合外部记账生态导入

## 数据导入

当前导入主要面向 `JSON` 数据。

导入前建议：

1. 先做快照
2. 确认目标身份
3. 导入后执行一次去重或重建检查

## 数据快照

快照是最重要的保险措施之一。

适合在以下操作前创建：

- 大规模导入
- 手动清理数据
- 更换规则文件
- 恢复历史账单
- 尝试全量重建

快照恢复后，你应该重新检查：

- 当前身份是否正确
- 账单数量是否符合预期
- 设置是否也回到了目标版本

## 更新与调试

### 更新

应用支持检查新版本，但是否升级仍建议由你自行判断。

### 调试

调试页更适合开发或排错使用，普通用户不建议频繁触碰恢复类操作。
