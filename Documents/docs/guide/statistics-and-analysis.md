# 统计分析

统计页适合做消费结构和趋势判断，而不是原始流水核对。

## 可用统计项

程序当前支持的主要统计有：

- 消费总览
- 每日趋势
- 分类分布
- 用餐时段分布
- 消费金额分桶
- 商户消费排行
- 忘刷卡统计
- 分类明细账单

## 首页统计与完整统计页的区别

首页更轻量，适合快速看：

- 今日消费
- 本月消费
- 趋势缩略图
- 分类占比缩略图

统计弹窗更完整，适合做详细分析：

- 更多时间范围
- 更多图表
- 可按分类下钻
- 可查看明细账单

## 时间范围建议

- `今日`：看当日消费与异常
- `本周 / 近 7 天`：看近期变化
- `本月`：看阶段消费习惯
- `季度 / 半年 / 一年`：看长期趋势
- `全部`：做历史总览

## 如何解读分类结果

分类统计依赖本地规则文件，因此它更适合：

- 看趋势
- 看占比
- 找高频消费点

而不是把它当作财务绝对真值。

如果某些交易分类不符合预期，通常不是账单丢失，而是规则命中方式需要调整。

## 忘刷卡统计

该统计用于识别特定模式的消费记录，帮助快速发现：

- 代刷嫌疑
- 忘刷卡补扣
- 与平时消费习惯明显不一致的记录

建议把它作为提醒信息，而不是单独作为最终结论。

## 统计前提

只有在以下条件满足时，统计才更有参考价值：

- 同步范围足够完整
- 合并账单没有明显重复
- 分类规则已正确加载

如果统计图明显异常，优先检查同步完整性，再检查规则文件。
