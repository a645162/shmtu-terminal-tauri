# 快速开始

这页只回答一个问题：第一次打开软件时，我该怎么尽快把账单同步出来。

## 1. 最终你会完成什么

第一次配置完成后，通常会走完这条最短路径：

1. 创建身份
2. 在身份下添加账号
3. 选择验证码模式
4. 进行一次增量或全量同步
5. 在首页、账单页、统计页查看结果

## 2. 开始前先准备好

建议先把下面几样东西准备好：

- 校园账号或学号
- 对应密码
- 你打算使用的验证码模式
- 是否需要把多个账号归到同一个身份

如果你只是想先确认软件能不能用，最稳妥的方式是先选 `手动验证码`。先跑通，再考虑换成 OCR。

## 3. 首次启动流程

### 第一步：先把程序打开

第一次打开时，程序会自动准备自己的本地数据和基础规则。你不需要手动建目录，也不需要先导入任何文件。

如果程序能正常进入主界面，就说明第一步已经完成。

### 第二步：先建一个“身份”

你可以把“身份”理解成一个数据分组。最简单的做法，就是先建一个属于你自己的身份。

建议按“人”或“用途”来命名：

- `我自己`
- `家人`
- `测试账号`

即使你只打算管理一个人，也建议建一个明确的身份，后面看统计、导出数据会更清楚。

### 第三步：在身份下面添加账号

在“身份与账号管理”里添加账号时，重点看这几项：

- `账号名称`：便于识别的展示名
- `账号 ID`：实际登录使用的账号标识
- `密码`：直接填真实密码即可
- `enable`：是否启用该账号
- `enable_update`：是否允许该账号参与同步

如果某个账号已经不用了，可以先保留，但把同步开关关掉。

### 第四步：先选验证码方式

你第一次用时，不需要追求“最自动化”，只需要先保证能成功同步。

可选方式如下：

- `manual`：手动输入验证码，最稳定，最适合首次验证环境。
- `remote_ocr`：走远程 TCP OCR 服务。
- `remote_ocr_http`：走远程 HTTP OCR 服务。
- `local_onnx`：本地加载 ONNX 模型识别验证码。

推荐顺序：

1. 首次使用先选 `manual`
2. 确认账单同步正常
3. 再切换到远程 OCR 或本地 OCR

### 第五步：做第一次同步

如果你只是想快速看到结果，可以先做一次 `增量同步`。

如果你希望把历史记录尽量补齐，第一次更推荐：

- `全量同步`
- 范围选 `最近半年` 或 `全部`

同步结束后，日常使用再改回 `增量同步`。

## 4. 先记住两条就够了

关于数据，普通用户先记住下面两条：

- 不要手动删除数据库和会话文件，除非你明确知道后果。
- 在做大规模导入、恢复、规则替换前，先创建一次快照。

## 5. 看到这些，说明你已经上手成功

完成第一次配置后，建议检查：

- 是否成功看到当前身份名称
- 账号列表是否完整
- 账单页是否能查询到记录
- 首页统计是否已刷新
- 设置中验证码模式是否符合预期
