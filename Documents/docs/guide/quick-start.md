# 快速开始

本节用于第一次安装和第一次打开程序。

## 1. 你会得到什么

首次完成配置后，程序通常会形成以下使用闭环：

1. 创建身份
2. 在身份下添加账号
3. 选择验证码模式
4. 进行一次增量或全量同步
5. 在首页、账单页、统计页查看结果

## 2. 启动前准备

建议先确认以下信息：

- 校园账号或学号
- 对应密码
- 你打算使用的验证码模式
- 是否需要把多个账号归到同一个身份

如果你没有 OCR 环境，最稳妥的方式是先用 `手动验证码` 跑通流程，再决定是否切到远程或本地 OCR。

## 3. 首次启动流程

### 第一步：启动程序

程序启动后会初始化本地数据目录，并尝试准备分类规则文件。

默认会涉及以下本地数据：

- 配置文件 `app_config.toml`
- 主数据库
- 身份、账号、会话与快照目录
- 分类规则与位置映射文件

### 第二步：创建身份

身份是最外层的数据隔离单元。建议按“人”或“用途”来建：

- `我自己`
- `家人`
- `测试账号`

如果你只管理一个人，也建议显式创建一个身份，后续导出、统计、切换都会更清晰。

### 第三步：添加账号

在“身份与账号管理”中添加账号时，建议确认：

- `账号名称`：便于识别的展示名
- `账号 ID`：实际登录使用的账号标识
- `密码`：程序会在存储前加密
- `enable`：是否启用该账号
- `enable_update`：是否允许该账号参与同步

如果账号已经毕业或停用，可以关闭同步开关，避免无意义请求。

### 第四步：选择验证码模式

可选模式如下：

- `manual`：手动输入验证码，最稳定，最适合首次验证环境。
- `remote_ocr`：走远程 TCP OCR 服务。
- `remote_ocr_http`：走远程 HTTP OCR 服务。
- `local_onnx`：本地加载 ONNX 模型识别验证码。

建议顺序：

1. 首次使用先选 `manual`
2. 确认账单同步正常
3. 再切换到远程 OCR 或本地 OCR

### 第五步：执行第一次同步

推荐第一次执行 `全量同步`，同步范围选择 `最近半年` 或 `全部`，这样更容易确认数据完整性。

同步完成后再改用 `增量同步` 作为日常模式。

## 4. 数据目录说明

程序会维护自己的数据目录。用户需要知道的重点只有两个：

- 不要手动删除数据库和会话文件，除非你明确知道后果。
- 在做大规模导入、恢复、规则替换前，先创建一次快照。

## 5. 首次检查清单

完成初始化后，建议逐项确认：

- 是否成功看到当前身份名称
- 账号列表是否完整
- 账单页是否能查询到记录
- 首页统计是否已刷新
- 设置中验证码模式是否符合预期
